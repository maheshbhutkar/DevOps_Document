This page provides the view of folder structure for different modules within CareID system.

 

 

Following table details the folder and the module it should contain.  

Parent Folder
Folder
Description
 CareIDThis is the root folder for CareID.
CareIDIntermediatePhaseThis folder will have the intermediate phase POC source code.
CareIDTechnicalSpikesThe source code built as part of technical spikes will be in this folder.
CareIDDevelopmentAll the modules within CareID system will be within this folder.
CareIDTestingThis folder will contain sub-folders for different aspects of testing such as Module testing, integration testing, penetration testing, load testing, etc.
DevelopmentAuthenticationMainThis folder represents the authentication module.
DevelopmentAuthenticationRoleAssertionThis folder represents the role selection module. Some of the key features provided are list of user roles, select the default role, assert role information in SAML, etc.
DevelopmentBusinessServicesThis folder will have sub-folders for different types of services. Some of the services could be SearchService, ReportingService, WorkflowService, etc.
DevelopmentCareIDWebThis represents the web layer for CareID system. Some of the key artefacts this folder will have are: html, js, images, css, controller and action classes.
DevelopmentReportingThis folder represents the Reporting module of CareID.
DevelopmentIntegrationEngineThe folder represents the integration module for CareID and might have sub-folders having modules such as Message Handler, Service Registry, Mediator, etc.
DevelopmentOrganisationWorkflowThis folder represents the workflow module for Organisation administration. Some of the key artefacts this folder will have are workflow definitions, rules file and classes for task services.
DevelopmentEndpointWorkflowThis folder represents endpoint registration module. Some of the key artefacts this folder will have are workflow definitions, rules file and classes for task services.
DevelopmentUserWorkflowThis folder represents the workflow module for user registration and access management and will also have access migration related workflows. Some of the key artefacts this folder will have are workflow definitions, rules file and classes for task services.
DevelopmentCredentialWorkflowThis folder represents credential management module. Some of the key artefacts this folder will have are workflow definitions, rules file and classes for task services.
DevelopmentFrameworkThis contains the sub-folders for all the infrastructure components such as, rule engine, workflow, audit, logging, etc.
Development --> FrameworkLoggingThis folder represents the logging module for CareID. This folder will have source code specific to performance logging and diagnostic logging.
Development --> FrameworkAuditThis folder represents the audit module for the CareID system.
Development --> FrameworkDomainModelAll the domain classes of CareID will be kept in this folders.
Development --> FrameworkFuzzySearchThis folder represents the module for fuzzy search.
Development --> FrameworkDataAccessServiceThis folder represents data access module for CareID system.
Development --> FrameworkRulesThis folder will contain the modules pertaining to rule engine.
Development --> Framework --> RulesRulesServiceThis folder represents the rule engine. This will contain the wrapper classes around the COTS product. 
Development --> Framework --> RulesRulesClientThis folder represents the client module for invoking rule engine (RuleService).
Development --> FrameworkWorkflowThis folder will contain the modules pertaining to workflow.
Development --> Framework --> WorkflowWorkflowServiceThis folder represents the workflow engine. This will contain the wrapper classes around the COTS product.
Development --> Framework --> WorkflowWorkflowClientThis folder represents the client module for invoking workflow services (WorkflowService).
